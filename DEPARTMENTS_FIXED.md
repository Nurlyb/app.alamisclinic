# Исправление создания отделений

## Проблема
При попытке создать отделение возникала ошибка 405 (Method Not Allowed).

## Причина
Next.js кэшировал старую версию маршрута `/api/departments`, где не был экспортирован метод POST.

## Решение

### 1. Добавлены методы в API клиент
Файл: `api/departments.api.ts`
- `create()` - создание отделения
- `update()` - обновление отделения  
- `delete()` - деактивация отделения

### 2. Очищен кэш Next.js
- Остановлен dev сервер
- Удалена папка `.next`
- Перезапущен dev сервер

### 3. Проверены маршруты
- ✅ `POST /api/departments` - создание отделения
- ✅ `GET /api/departments` - список отделений
- ✅ `GET /api/departments/[id]` - получение по ID
- ✅ `PUT /api/departments/[id]` - обновление
- ✅ `DELETE /api/departments/[id]` - деактивация

## Тестирование
Теперь можно создавать отделения через форму в настройках:
1. Перейти в "Настройки" → вкладка "Отделения"
2. Нажать "Добавить отделение"
3. Ввести название (например: "Проктология")
4. Нажать "Создать"

## Статус
✅ Проблема решена. Сервер перезапущен с чистым кэшем.
