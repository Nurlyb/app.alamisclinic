## Основные команды Git

### Проверить статус изменений
```bash
git status
```

### Добавить все изменения
```bash
git add .
```

### Сделать коммит
```bash
git commit -m "Описание изменений"
```

### Отправить на GitHub
```bash
git push
```

### Получить последние изменения
```bash
git pull
```

### Посмотреть историю коммитов
```bash
git log --oneline
```

## Рабочий процесс

### 1. После внесения изменений:
```bash
git add .
git commit -m "Добавлена функция X"
git push
```

### 2. Перед началом работы (если работаете с нескольких устройств):
```bash
git pull
```

### 3. Создание новой ветки для экспериментов:
```bash
git checkout -b feature/новая-функция
# Работаете над функцией
git add .
git commit -m "Реализована новая функция"
git push -u origin feature/новая-функция
```

### 4. Слияние ветки с main:
```bash
git checkout main
git merge feature/новая-функция
git push
```

## Полезные команды

### Отменить последний коммит (но сохранить изменения)
```bash
git reset --soft HEAD~1
```

### Отменить изменения в файле
```bash
git checkout -- имя_файла
```

### Посмотреть изменения
```bash
git diff
```

### Посмотреть изменения в конкретном файле
```bash
git diff имя_файла
```

### Создать тег (версию)
```bash
git tag -a v1.0.0 -m "Версия 1.0.0"
git push --tags
```

## Настройка для нового компьютера

Если вы хотите клонировать репозиторий на другой компьютер:

```bash
# 1. Клонировать репозиторий
git clone https://github.com/Nurlyb/app.alamisclinic.git

# 2. Перейти в папку
cd app.alamisclinic

# 3. Установить зависимости
npm install

# 4. Создать .env файл (скопировать из .env.example)
cp .env.example .env

# 5. Настроить .env с вашими данными

# 6. Запустить миграции БД
npx prisma migrate dev

# 7. Запустить приложение
npm run dev
```

## Безопасность

### ⚠️ ВАЖНО: Никогда не коммитьте:
- Файлы .env с паролями и ключами
- node_modules/
- Файлы с персональными данными пациентов
- Секретные ключи API

### Если случайно закоммитили секретный файл:

```bash
# 1. Удалить файл из git (но оставить локально)
git rm --cached .env

# 2. Добавить в .gitignore
echo ".env" >> .gitignore

# 3. Закоммитить изменения
git add .gitignore
git commit -m "Удален .env из репозитория"
git push

# 4. ВАЖНО: Сменить все пароли и ключи, которые были в файле!
```

## Резервное копирование

Репозиторий на GitHub - это уже резервная копия вашего кода. Но для полной безопасности:

1. Регулярно делайте коммиты и push
2. Создавайте теги для важных версий
3. Делайте резервные копии базы данных отдельно
4. Храните .env файл в безопасном месте (не в git!)
