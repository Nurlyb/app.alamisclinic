# Анализ системы прав доступа

## Проблема
Регистратор получает ошибку "недостаточно прав" при попытке создать запись.

## Анализ системы прав

### 1. Схема базы данных (prisma/schema.prisma)
✅ **Корректно**: Роль RECEPTIONIST определена в enum Role

### 2. Создание пользователей (prisma/seed.ts)
✅ **Корректно**: Регистратор создается с ролью RECEPTIONIST:
```typescript
const receptionist1 = await prisma.user.create({
  data: {
    name: 'Мария Иванова',
    role: 'RECEPTIONIST',
    // ...
  },
});
```

### 3. Процесс логина (app/api/auth/login/route.ts)
✅ **Корректно**: Роль из БД попадает в JWT токен:
```typescript
const tokens = generateTokenPair({
  userId: user.id,
  role: user.role, // Роль из БД
  name: user.name,
  departmentId: user.departmentId || undefined,
});
```

### 4. Система RBAC (lib/auth/rbac.ts)
✅ **Исправлено**: Роль RECEPTIONIST имеет право appointments:create:
```typescript
RECEPTIONIST: [
  'appointments:view:all',
  'appointments:create', // ✅ Право есть
  'appointments:arrive',
  'appointments:update',
  // ...
],
```

**🔧 Исправлена проблема с типами**: Обновлен тип Role для совместимости с Prisma.

### 5. API создания записи (app/api/appointments/route.ts)
✅ **Корректно**: Endpoint защищен правом appointments:create:
```typescript
export const POST = withAuth(
  async (request, user) => {
    // логика создания записи
  },
  'appointments:create' // ✅ Требуется это право
);
```

### 6. Middleware проверки прав (lib/auth/middleware.ts)
✅ **Корректно**: Проверяет права через hasAnyPermission:
```typescript
const hasAccess = hasAnyPermission(user.role, requiredPermissions);
```

### 7. Frontend проверка прав (hooks/usePermissions.ts)
✅ **Корректно**: Использует ту же систему RBAC:
```typescript
const can = (permission: Permission): boolean => {
  if (!user) return false;
  return hasPermission(user.role, permission);
};
```

## Возможные причины проблемы

### 1. Кэширование данных пользователя
- JWT токен содержит старую роль (до добавления права)
- Данные пользователя в localStorage устарели

### 2. Несоответствие типов
- **🔧 ИСПРАВЛЕНО**: Конфликт между типами Role в types/index.ts и Prisma

### 3. Проблемы с сессией
- Пользователь не перелогинился после изменения прав
- Токен истек

## Решение

### 1. Добавлены инструменты отладки
- **Панель отладки прав** в правом нижнем углу
- **4 кнопки диагностики**:
  - 🔄 Обновить данные пользователя
  - 🔍 Проверить права на сервере и в БД
  - 🧪 Прямой тест прав (новая кнопка)
  - 🔑 Обновить токен с актуальной ролью

### 2. Исправлены типы в RBAC
- Обновлен тип Role для совместимости с Prisma
- Улучшено логирование проверки прав

### 3. Добавлены API endpoints для отладки
- `/api/debug/permissions` - проверка прав
- `/api/debug/user` - сравнение токена и БД
- `/api/auth/refresh-role` - обновление токена

## Инструкции для пользователя

1. **Откройте страницу расписания**
2. **Нажмите кнопку 🧪** для прямого теста прав
3. **Если тест показывает НЕТ** - нажмите 🔑 для обновления токена
4. **Если проблема сохраняется** - перелогиньтесь
