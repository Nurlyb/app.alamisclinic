# Добавлены фильтры для ассистентов

## Что изменено

Ассистенты теперь могут просматривать расписание и операции всех докторов, используя фильтры по отделениям и докторам.

## Изменения в расписании

### 1. Страница расписания (`app/schedule/page.tsx`)

**Изменения**:
- Убрана привязка ассистента только к своему доктору
- Ассистенты теперь видят все записи
- Могут фильтровать по отделениям и докторам
- Обновлен текст подсказки: "Вы можете просматривать расписание всех докторов"

**Логика фильтрации**:
```typescript
// Ассистент видит все записи (может фильтровать через отделения и докторов)
if (user?.role === 'ASSISTANT') {
  return true;
}
```

### 2. API appointments (`app/api/appointments/route.ts`)

**Изменения**:
- Убрана проверка `assistingDoctorId` для ассистентов
- Ассистенты получают все записи
- Фильтрация происходит на фронтенде через параметры `doctorId` и `departmentId`

**Код**:
```typescript
// Ассистенты видят все записи (могут фильтровать через параметры)
else if (user.role === 'ASSISTANT') {
  // Ассистенты имеют доступ ко всем записям
  // Фильтрация происходит через параметры doctorId и departmentId
}
```

## Изменения в календаре операций

### 1. Страница календаря (`app/operations-calendar/page.tsx`)

**Добавлено**:
- Импорты для Select компонентов
- Импорты API для отделений и докторов
- State для выбранного отделения и доктора
- Загрузка списка отделений (только для ассистентов)
- Загрузка списка докторов по отделению (только для ассистентов)
- Секция с фильтрами (отображается только для ассистентов)
- Передача `doctorId` в API запрос

**UI фильтров**:
```tsx
{user?.role === 'ASSISTANT' && (
  <div className="bg-white rounded-lg shadow-sm p-4">
    <div className="flex items-center gap-4">
      {/* Выбор отделения */}
      <Select value={selectedDepartmentId} onValueChange={...}>
        <SelectItem value="all">Все отделения</SelectItem>
        {departments.map(...)}
      </Select>
      
      {/* Выбор доктора */}
      <Select value={selectedDoctorId} onValueChange={...}>
        <SelectItem value="all">Все доктора</SelectItem>
        {doctors.map(...)}
      </Select>
    </div>
  </div>
)}
```

**Обновлен текст**: "Вы можете просматривать операции всех докторов"

### 2. API календаря операций (`app/api/doctor-service-assignments/calendar/route.ts`)

**Изменения**:
- Добавлен параметр `doctorId` в query string
- Для докторов: показываются только свои операции
- Для ассистентов: 
  - Если `doctorId` указан и не 'all' - показываются операции этого доктора
  - Если `doctorId` не указан или 'all' - показываются операции всех докторов

**Логика**:
```typescript
if (user.role === 'DOCTOR') {
  doctorId = user.userId; // Только свои
} else if (user.role === 'ASSISTANT') {
  if (doctorIdParam && doctorIdParam !== 'all') {
    doctorId = doctorIdParam; // Выбранный доктор
  }
  // Если не указан - показываем всех
}
```

## Как работает

### Расписание
1. Ассистент заходит в "Расписание"
2. Видит все записи всех докторов
3. Может выбрать отделение → загружаются доктора этого отделения
4. Может выбрать конкретного доктора → показываются только его записи
5. Может выбрать "Все отделения" / "Все доктора" → показываются все записи

### Календарь операций
1. Ассистент заходит в "Календарь операций"
2. По умолчанию видит операции всех докторов
3. Может выбрать отделение → загружаются доктора этого отделения
4. Может выбрать конкретного доктора → показываются только его операции
5. Может вернуться к "Все отделения" / "Все доктора" → показываются все операции

## Права доступа

Используются существующие права:
- `appointments:view:all` - ассистенты имеют это право
- Позволяет видеть все записи и операции
- Фильтрация происходит на уровне UI

## Преимущества

1. **Гибкость**: Ассистент может помогать любому доктору, не только своему
2. **Удобство**: Быстрое переключение между докторами через фильтры
3. **Обзор**: Возможность видеть общую картину по всей клинике
4. **Совместимость**: Поле `assistingDoctorId` сохранено для будущего использования
