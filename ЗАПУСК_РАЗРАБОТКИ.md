# Инструкция по запуску и тестированию

## Применение изменений в базе данных

### 1. Применить миграцию
```bash
npx prisma migrate dev --name add_doctor_features
```

### 2. Сгенерировать Prisma Client
```bash
npx prisma generate
```

### 3. Проверить схему
```bash
npx prisma validate
```

## Запуск приложения

### Режим разработки
```bash
npm run dev
```

Приложение будет доступно по адресу: http://localhost:3000

## Тестирование нового функционала

### 1. Создание доктора

1. Войдите как владелец (OWNER)
2. Перейдите в "Пользователи"
3. Нажмите "Создать пользователя"
4. Заполните:
   - Имя: Доктор Иванов
   - Роль: Доктор
   - Отделение: Выберите любое
   - Телефон: +77001234567
   - Пароль: password123
   - Цвет бейджа: Любой
5. Сохраните

### 2. Создание ассистента

1. В "Пользователи" нажмите "Создать пользователя"
2. Заполните:
   - Имя: Ассистент Петров
   - Роль: Ассистент
   - Доктор для ассистирования: Выберите созданного доктора
   - Телефон: +77001234568
   - Пароль: password123
   - Цвет бейджа: Любой
3. Сохраните

### 3. Добавление услуг доктором

1. Выйдите и войдите как доктор (+77001234567 / password123)
2. Вы попадете на "Панель доктора"
3. Перейдите на вкладку "Мои услуги"
4. Нажмите "Добавить услугу"
5. Выберите услугу из списка (например, "Консультация")
6. Укажите индивидуальную цену: 15000
7. Сохраните
8. Добавьте еще несколько услуг

### 4. Создание записи к доктору

1. Войдите как оператор или владелец
2. Перейдите в "Расписание"
3. Создайте запись:
   - Пациент: Выберите или создайте нового
   - Доктор: Выберите созданного доктора
   - Услуга: Выберите любую
   - Дата и время: Сегодня
4. Сохраните

### 5. Работа с медицинской карточкой (доктор)

1. Войдите как доктор
2. На вкладке "Мои записи" найдите созданную запись
3. Нажмите "Открыть карточку"
4. Заполните:
   - Жалобы: "Боль в спине"
   - Диагноз: "Остеохондроз"
   - Код МКБ-10: M42
   - Рецепт: "Диклофенак 75мг 2 раза в день"
   - Рекомендации: "Покой, физиотерапия"
   - Витальные показатели:
     - Вес: 75
     - Рост: 180
     - Давление: 120/80
     - Температура: 36.6
5. Сохраните

### 6. Просмотр пациентов (доктор)

1. Перейдите на вкладку "Мои пациенты"
2. Вы увидите список пациентов, у которых были записи к вам
3. Нажмите на пациента
4. Справа отобразится история его приёмов
5. Нажмите "Карточка" на любом приёме для просмотра/редактирования

### 7. Проверка доступа ассистента

1. Выйдите и войдите как ассистент (+77001234568 / password123)
2. Вы попадете на "Панель ассистента"
3. Проверьте:
   - Вкладка "Мои записи" - должны быть видны записи доктора
   - Вкладка "Мои пациенты" - должны быть видны пациенты доктора
   - Вкладка "Мои услуги" - должны быть видны услуги доктора (без возможности редактирования)
4. Откройте карточку пациента и убедитесь, что можете её редактировать

### 8. Создание направления (доктор)

1. Войдите как доктор
2. Откройте запись пациента
3. В медицинской карточке можно создать направление к другому специалисту
4. Направление появится в системе

## Проверка прав доступа

### Доктор должен видеть:
- ✅ Только свои записи
- ✅ Только своих пациентов
- ✅ Свои услуги с возможностью редактирования
- ✅ Медицинские карточки своих пациентов

### Доктор НЕ должен видеть:
- ❌ Записи других докторов
- ❌ Пациентов других докторов
- ❌ Финансовую информацию (кроме своей зарплаты)
- ❌ Настройки системы

### Ассистент должен видеть:
- ✅ Все записи своего доктора
- ✅ Всех пациентов своего доктора
- ✅ Услуги своего доктора (только просмотр)
- ✅ Медицинские карточки пациентов доктора

### Ассистент НЕ должен видеть:
- ❌ Записи других докторов
- ❌ Возможность редактировать услуги доктора
- ❌ Финансовую информацию

## Проверка API

### Получить услуги доктора
```bash
curl -X GET "http://localhost:3000/api/doctor-services?doctorId=DOCTOR_ID" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### Создать услугу доктора
```bash
curl -X POST "http://localhost:3000/api/doctor-services" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "serviceId": "SERVICE_ID",
    "customPrice": 50000
  }'
```

### Обновить услугу доктора
```bash
curl -X PATCH "http://localhost:3000/api/doctor-services/SERVICE_ID" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "customPrice": 60000
  }'
```

## Проверка логирования

1. Войдите как владелец
2. Перейдите в "Журнал действий"
3. Проверьте, что все действия докторов и ассистентов логируются:
   - Создание услуг
   - Заполнение медицинских карточек
   - Создание направлений

## Возможные проблемы и решения

### Ошибка миграции
```bash
# Сбросить базу данных (ВНИМАНИЕ: удалит все данные)
npx prisma migrate reset

# Применить миграции заново
npx prisma migrate dev
```

### Ошибка типов TypeScript
```bash
# Перегенерировать Prisma Client
npx prisma generate

# Перезапустить TypeScript сервер в IDE
```

### Ассистент не видит доктора
- Проверьте, что поле `assistingDoctorId` заполнено в базе данных
- Проверьте, что доктор активен (`isActive = true`)

### Услуги не отображаются
- Проверьте, что услуги активны (`isActive = true`)
- Проверьте права доступа пользователя

## Следующие шаги

2. Убедитесь, что права доступа работают корректно
3. Проверьте логирование всех действий
4. Протестируйте на мобильных устройствах
5. Когда всё готово - скажите, и мы задеплоим на сервер
