## Технические детали

### Права доступа (RBAC)
Используются существующие права:
- `appointments:view:own` - для докторов
- `appointments:view:all` - для ассистентов
- Ассистенты имеют оба права для доступа к записям своего доктора

### База данных
Используется существующая модель `DoctorServiceAssignment`:
- `scheduledDate` - дата и время операции
- `status` - статус (PLANNED, COMPLETED, CANCELLED)
- `doctorId` - ID доктора
- `patientId` - ID пациента
- `serviceId` - ID услуги
- `price` - стоимость
- `notes` - примечания

### Связь с существующим функционалом
Операции создаются в разделе "Мои пациенты":
1. Доктор открывает карточку пациента
2. Нажимает "Назначить операцию"
3. Выбирает услугу, указывает цену, дату/время
4. Операция сохраняется в БД
5. Отображается в "Календаре операций"
