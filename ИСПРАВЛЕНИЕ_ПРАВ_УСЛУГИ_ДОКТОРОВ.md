# Исправлены права доступа для услуг докторов

## Проблема
Владелец получал ошибку 403 (Forbidden) при попытке создать, обновить или удалить услугу доктора.

## Причина
API endpoints для услуг докторов требовали права, которых не было у владельца:
- `services:create` - для создания
- `services:update` - для обновления  
- `services:delete` - для удаления

Но у владельца в RBAC было только право `services:manage`.

## Решение
Изменены проверки прав в API endpoints на `services:manage`, которое есть у владельца.

## Измененные файлы

### 1. `app/api/doctor-services/route.ts`
**Было**: `'services:create'`
**Стало**: `'services:manage'`

```typescript
export const POST = withAuth(
  async (request, user) => {
    // ... код создания услуги
  },
  'services:manage' // Изменено с 'services:create'
);
```

### 2. `app/api/doctor-services/[id]/route.ts`
**Было**: `'services:update'` и `'services:delete'`
**Стало**: `'services:manage'` для обоих

```typescript
export const PUT = withAuth(
  async (request, user, { params }) => {
    // ... код обновления услуги
  },
  'services:manage' // Изменено с 'services:update'
);

export const DELETE = withAuth(
  async (request, user, { params }) => {
    // ... код удаления услуги
  },
  'services:manage' // Изменено с 'services:delete'
);
```

## Права владельца (RBAC)
Владелец имеет право `services:manage`, которое теперь используется для всех операций с услугами докторов:
- Создание (POST)
- Обновление (PUT)
- Удаление (DELETE)
- Просмотр (GET) - доступен всем с правами `appointments:view:all` или `appointments:view:own`

## Результат
✅ Владелец теперь может:
- Создавать новые услуги докторов
- Редактировать существующие услуги
- Удалять (деактивировать) услуги

## Тестирование
1. Войти как владелец (+77001234567 / clinic123)
2. Перейти в Настройки → вкладка "Услуги докторов"
3. Нажать "Добавить услугу"
4. Заполнить название и описание
5. Сохранить
6. Услуга должна успешно создаться без ошибки 403

## Альтернативное решение
Можно было бы добавить права `services:create`, `services:update`, `services:delete` владельцу в RBAC, но использование единого права `services:manage` более логично и соответствует существующей структуре прав.
