# Исправлено сохранение данных пациента

## Проблемы
1. Изменения данных пациента не сохранялись в базе данных
2. Доктора и ассистенты получали ошибку 403 при попытке обновить данные пациента
3. ИИН не сохранялся при обновлении

## Причины

### 1. Вложенные формы
Форма редактирования пациента была вложена в основную форму диагноза. В HTML нельзя вкладывать формы друг в друга, что приводило к некорректной отправке данных.

### 2. Отсутствие прав
У докторов (DOCTOR) и ассистентов (ASSISTANT) не было права `patients:update` в RBAC.

### 3. ИИН не обрабатывался
API endpoint не обрабатывал поле `iin` при обновлении пациента.

## Решения

### 1. Исправлена структура форм (`components/schedule/MedicalRecordForm.tsx`)

**Было**:
```tsx
<form onSubmit={handleSubmit(onSubmit)}>
  {/* Карточка пациента */}
  <form onSubmit={handleSubmitPatient(onSubmitPatient)}>
    {/* Поля редактирования */}
    <Button type="submit">Сохранить</Button>
  </form>
</form>
```

**Стало**:
```tsx
<form onSubmit={handleSubmit(onSubmit)}>
  {/* Карточка пациента */}
  <div>
    {/* Поля редактирования */}
    <Button type="button" onClick={handleSubmitPatient(onSubmitPatient)}>
      Сохранить
    </Button>
  </div>
</form>
```

Изменения:
- Заменен `<form>` на `<div>` для формы редактирования пациента
- Изменен `type="submit"` на `type="button"` для кнопки
- Добавлен `onClick={handleSubmitPatient(onSubmitPatient)}` для обработки клика

### 2. Добавлены права в RBAC (`lib/auth/rbac.ts`)

Добавлено право `patients:update` для:
- **DOCTOR**: может редактировать данные пациентов
- **ASSISTANT**: может редактировать данные пациентов

```typescript
ASSISTANT: [
  // ...
  'patients:view',
  'patients:update', // Добавлено
  'patients:files:upload',
  // ...
],

DOCTOR: [
  // ...
  'patients:view',
  'patients:update', // Добавлено
  'patients:files:view',
  // ...
],
```

### 3. Добавлена обработка ИИН (`app/api/patients/[id]/route.ts`)

**Было**:
```typescript
const updatedPatient = await prisma.patient.update({
  where: { id },
  data: {
    ...(data.fullName && { fullName: data.fullName }),
    ...(data.dob && { dob: new Date(data.dob) }),
    // ИИН не обрабатывался
    ...(data.gender && { gender: data.gender }),
    ...(data.phone && { phone: data.phone }),
    ...(data.address !== undefined && { address: data.address }),
    ...(data.source && { source: data.source }),
  },
});
```

**Стало**:
```typescript
const updatedPatient = await prisma.patient.update({
  where: { id },
  data: {
    ...(data.fullName && { fullName: data.fullName }),
    ...(data.iin !== undefined && { iin: data.iin || null }), // Добавлено
    ...(data.dob && { dob: new Date(data.dob) }),
    ...(data.gender && { gender: data.gender }),
    ...(data.phone && { phone: data.phone }),
    ...(data.address !== undefined && { address: data.address }),
    ...(data.source && { source: data.source }),
  },
});
```

## Результат

✅ Форма редактирования пациента работает корректно
✅ Доктора и ассистенты могут обновлять данные пациентов
✅ Все поля сохраняются в базу данных:
  - ФИО
  - ИИН
  - Телефон
  - Дата рождения
  - Пол
  - Адрес

✅ API возвращает 200 OK вместо 403 Forbidden
✅ Данные обновляются в реальном времени
✅ Карточка пациента автоматически обновляется после сохранения

## Тестирование

1. Войти как доктор или ассистент
2. Открыть форму "Записать диагноз"
3. Нажать "Редактировать" в карточке пациента
4. Изменить любые данные (ФИО, ИИН, телефон и т.д.)
5. Нажать "Сохранить изменения"
6. Убедиться что:
   - Показывается уведомление "Данные пациента обновлены"
   - Карточка возвращается в режим просмотра
   - Изменения отображаются в карточке
   - Данные сохранены в базе данных
