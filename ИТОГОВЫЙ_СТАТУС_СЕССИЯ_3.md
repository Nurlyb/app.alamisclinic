## 📊 Технические детали

### Права доступа (RBAC)
- `appointments:view:all` - ассистенты имеют это право
- `appointments:view:own` - для докторов
- Фильтрация на уровне API и UI

### База данных
- Модель `DoctorServiceAssignment` для операций
- Поле `assistingDoctorId` сохранено для будущего использования
- Поле `scheduledDate` для планирования операций

### API Endpoints
1. `GET /api/appointments?date=YYYY-MM-DD` - записи на прием
2. `GET /api/doctor-service-assignments/calendar?date=YYYY-MM-DD&doctorId=xxx` - операции
3. `GET /api/users/[id]` - информация о пользователе
4. `GET /api/departments` - список отделений
5. `GET /api/users?role=DOCTOR&departmentId=xxx` - доктора по отделению

## 🎯 Функциональность

### Для докторов
- Видят только свои записи в расписании
- Видят только свои операции в календаре
- При клике на запись → переход к карточке пациента
- Не имеют фильтров (видят только свое)

### Для ассистентов
- Видят все записи в расписании
- Видят все операции в календаре
- Могут фильтровать по отделениям и докторам
- При клике на запись → переход к карточке пациента
- Полная гибкость в работе с любым доктором

### Для других ролей
- Владелец: видит всё
- Оператор: видит только свои записи
- Регистратор: видит всё, может отмечать приход
