# Календарь операций для докторов и ассистентов

## Что добавлено

Создан новый раздел "Календарь операций" для докторов и ассистентов, где они могут видеть запланированные операции по датам.

## Файлы

### 1. API Endpoint
**Файл**: `app/api/doctor-service-assignments/calendar/route.ts`

**Endpoint**: `GET /api/doctor-service-assignments/calendar?date=YYYY-MM-DD`

**Функционал**:
- Получает операции по выбранной дате
- Доктора видят свои операции
- Ассистенты видят операции своего доктора (через `assistingDoctorId`)
- Возвращает операции с информацией о пациенте и услуге
- Форматирует дату и время для отображения

**Права доступа**: `appointments:view:own`, `appointments:view:all`

### 2. Страница календаря
**Файл**: `app/operations-calendar/page.tsx`

**Функционал**:
- Календарная сетка с временными слотами (8:00 - 20:00, каждые 30 минут)
- Навигация по датам (вперед/назад, выбор даты, кнопка "Сегодня")
- Карточки операций с информацией:
  - Статус (Запланировано/Выполнено/Отменено)
  - Пациент (ФИО, телефон)
  - Название операции
  - Стоимость
- Клик по операции открывает детальную информацию
- Операции без указанного времени показываются отдельно внизу
- Прошедшие временные слоты затемнены

**Статусы операций**:
- `PLANNED` - Запланировано (синий)
- `COMPLETED` - Выполнено (зеленый)
- `CANCELLED` - Отменено (красный)

### 3. Навигация
**Файл**: `components/layout/Sidebar.tsx`

Добавлена ссылка "Календарь операций" с иконкой ножниц (Scissors) для:
- Докторов (DOCTOR)
- Ассистентов (ASSISTANT)

## Как работает

1. **Доктор заходит в "Календарь операций"**:
   - Видит все свои запланированные операции
   - Может переключаться между датами
   - Может кликнуть на операцию для просмотра деталей

2. **Ассистент заходит в "Календарь операций"**:
   - Видит операции своего доктора (к которому привязан через `assistingDoctorId`)
   - Имеет те же возможности, что и доктор

3. **Отображение операций**:
   - Операции группируются по времени
   - Каждая операция показывается в виде карточки
   - Цвет карточки зависит от статуса
   - Прошедшие слоты затемнены

4. **Детали операции**:
   - Открываются в боковой панели (Sheet)
   - Показывают полную информацию:
     - Статус с иконкой
     - Информация о пациенте
     - Название операции
     - Стоимость
     - Примечания (если есть)
     - Дата выполнения (если выполнено)

## Связь с существующим функционалом

Операции создаются в разделе "Мои пациенты" через форму "Назначить операцию":
- Доктор выбирает услугу из своего списка услуг
- Указывает цену
- Может указать дату и время операции (`scheduledDate`)
- Может добавить примечания

Эти операции затем отображаются в "Календаре операций" по датам.

## Права доступа

Используются существующие права:
- `appointments:view:own` - для докторов (свои операции)
- `appointments:view:all` - для ассистентов (операции своего доктора)

## UI/UX особенности

- Адаптивная сетка (1-3 колонки в зависимости от ширины экрана)
- Цветовая индикация статусов
- Иконки для быстрого распознавания
- Затемнение прошедших слотов
- Отдельная секция для операций без времени
- Плавные анимации при наведении
- Responsive дизайн для мобильных устройств
