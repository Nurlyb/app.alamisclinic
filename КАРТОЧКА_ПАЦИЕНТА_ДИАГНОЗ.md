# Обновлена форма записи диагноза с полной карточкой пациента

## Что изменено

При нажатии на кнопку "Записать диагноз" теперь открывается расширенная карточка пациента с полной информацией.

## Изменения

### Файл: `components/schedule/MedicalRecordForm.tsx`

**Добавлено**:
1. Загрузка полной информации о пациенте через API
2. Красивая карточка пациента с градиентом
3. Отображение всех данных пациента в отдельных блоках

**Отображаемая информация**:

### 1. Основная информация
- **ФИО** - полное имя пациента
- **ИИН** - индивидуальный идентификационный номер (если указан)
- **Телефон** - контактный номер
- **Дата рождения** - в формате "1 января 1990"
- **Пол** - Мужской/Женский
- **Адрес** - место проживания

### 2. Дополнительная информация
- **Статус "Чёрный список"** - если пациент в чёрном списке, показывается красный бейдж
- **Задолженность** - если есть долг, показывается в красном блоке с суммой

### Цветовая схема
- **Карточка**: Градиент от голубого к индиго
- **Блоки данных**: Белый фон с голубой рамкой
- **Чёрный список**: Красный бейдж
- **Задолженность**: Красный блок с предупреждением

### Иконки
- 👤 User - для ФИО и пола
- 💳 CreditCard - для ИИН
- 📞 Phone - для телефона
- 📅 Calendar - для даты рождения
- 📍 MapPin - для адреса

## Адаптивность

- **Desktop**: 2 колонки (grid-cols-2)
- **Mobile**: 1 колонка (grid-cols-1)
- Все блоки автоматически перестраиваются

## Логика отображения

### Обработка пустых значений
- **ИИН**: "Не указан" если null
- **Дата рождения**: "Не указана" если null
- **Пол**: "Не указан" если null, иначе "Мужской"/"Женский"
- **Адрес**: "Не указан" если null

### Условное отображение
- **Бейдж "Чёрный список"**: показывается только если `blacklist === true`
- **Блок задолженности**: показывается только если `debt > 0`

## API запрос

```typescript
GET /api/patients/${patientId}
```

**Возвращает**:
```typescript
{
  id: string;
  fullName: string;
  iin: string | null;
  dob: string | null;
  gender: 'MALE' | 'FEMALE' | null;
  phone: string;
  address: string | null;
  blacklist: boolean;
  debt: number;
}
```

## Где используется

1. **Расписание** (`app/schedule/page.tsx`)
   - Кнопка "📝 Записать диагноз" для докторов
   - Открывается в Dialog

2. **Мои пациенты** (`app/doctor-patients/page.tsx`)
   - Кнопка "Записать диагноз" в карточке пациента
   - Открывается в Dialog

## Преимущества

1. **Полная информация** - доктор видит все данные пациента перед записью диагноза
2. **Удобство** - не нужно переключаться между страницами
3. **Визуальная ясность** - информация структурирована и легко читается
4. **Предупреждения** - сразу видно если пациент в чёрном списке или имеет долг
5. **Профессионально** - красивый дизайн повышает доверие к системе

## Пример использования

1. Доктор открывает расписание
2. Видит пациента со статусом "Прибыл"
3. Нажимает "📝 Записать диагноз"
4. Открывается форма с полной карточкой пациента:
   - Видит ФИО, ИИН, телефон
   - Видит дату рождения и возраст
   - Видит адрес для контекста
   - Видит предупреждение о задолженности (если есть)
5. Записывает диагноз и рекомендации
6. Сохраняет запись

## Технические детали

- Использует `useQuery` для загрузки данных пациента
- Автоматическая загрузка при открытии формы
- Кэширование данных через React Query
- Форматирование даты через `date-fns` с русской локалью
- Responsive дизайн через Tailwind CSS Grid
