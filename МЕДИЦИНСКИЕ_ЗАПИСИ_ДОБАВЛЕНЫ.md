# Медицинские записи для докторов - Добавлено

## Что сделано

### 1. Интеграция формы медицинских записей в расписание

Добавлена возможность для докторов записывать диагнозы и рекомендации для своих пациентов прямо из расписания.

### 2. Кнопка "Записать диагноз"

- Появляется только для докторов
- Только для своих пациентов (где doctorId === user.id)
- Только когда пациент прибыл (status === 'ARRIVED')
- Не показывается для прошедших записей

### 3. Модальное окно с формой

- Открывается при нажатии на кнопку "📝 Записать диагноз"
- Показывает информацию о пациенте
- Поля для ввода:
  - Диагноз (необязательно)
  - Примечания / Рекомендации (необязательно)
- Кнопка "Сохранить запись"

### 4. API endpoint

Уже существует: `POST /api/medical-records`

- Создает новую медицинскую запись
- Если запись уже существует - обновляет её
- Доктор может создавать записи только для своих пациентов
- Сохраняет в таблицу `medical_records`:
  - appointmentId
  - patientId
  - doctorId
  - diagnosis
  - notes

## Файлы изменены

1. `app/schedule/page.tsx` - добавлена кнопка и модальное окно
2. `components/schedule/MedicalRecordForm.tsx` - форма для записи диагноза (уже существовала)
3. `app/api/medical-records/route.ts` - API endpoint (уже существовал)

## Как использовать

1. Доктор заходит в расписание
2. Видит только свои записи
3. Когда пациент прибыл (статус "Прибыл")
4. Открывает детали записи
5. Нажимает кнопку "📝 Записать диагноз"
6. Заполняет форму
7. Нажимает "Сохранить запись"
8. Запись сохраняется в базу данных
