# Операции для регистратора - Реализовано

## Что сделано

### 1. API Endpoints

#### GET /api/doctor-service-assignments/receptionist
- Получение операций со статусом `PLANNED` (не оплаченные) на выбранную дату
- Доступно только для регистратора (право `payments:create`)
- Возвращает операции с информацией о пациенте и услуге
- Группировка по времени

#### POST /api/doctor-service-assignments/[id]/payment
- Прием оплаты за операцию
- Обновляет статус операции на `COMPLETED`
- Сохраняет информацию об оплате в примечаниях
- Доступно только для регистратора (право `payments:create`)

### 2. UI Компоненты

#### OperationPaymentModal
- Модальное окно для приема оплаты за операцию
- Показывает информацию о пациенте, операции и стоимости
- Поддерживает 3 способа оплаты:
  - Наличные (с расчетом сдачи)
  - Kaspi
  - Карта
- Валидация суммы оплаты

#### Обновленная страница расписания (app/schedule/page.tsx)
- Для регистратора добавлены вкладки:
  - 📅 Записи на прием (существующий функционал)
  - ✂️ Операции (новый функционал)
- Операции отображаются в календарном виде по времени
- Клик по операции открывает модальное окно оплаты
- Операции без времени показываются отдельно

### 3. Компоненты UI

#### RadioGroup (components/ui/radio-group.tsx)
- Новый компонент для выбора способа оплаты
- Использует Radix UI

## Как это работает

### Для регистратора:

1. Открыть страницу "Расписание" (`/schedule`)
2. Переключиться на вкладку "✂️ Операции"
3. Выбрать дату
4. Увидеть список операций, запланированных на эту дату
5. Кликнуть на операцию
6. В модальном окне:
   - Проверить информацию о пациенте и операции
   - Выбрать способ оплаты
   - Для наличных - ввести полученную сумму (автоматически рассчитается сдача)
   - Нажать "Принять оплату"
7. Операция исчезнет из списка (статус изменится на COMPLETED)

### Для доктора:

1. Доктор назначает операцию пациенту через свою панель
2. Указывает дату, время и цену операции
3. Операция появляется в календаре операций
4. Операция также появляется у регистратора для приема оплаты

## Технические детали

### Статусы операций:
- `PLANNED` - запланировано, ожидает оплаты (видно регистратору)
- `COMPLETED` - выполнено и оплачено (не видно регистратору)
- `CANCELLED` - отменено

### Права доступа:
- Регистратор имеет право `payments:create`
- Это право используется для доступа к операциям и их оплате

### Информация об оплате:
- Сохраняется в поле `notes` операции
- Формат: "Оплата: [сумма] ₸ ([способ]) - [дата и время]"
- Дата завершения сохраняется в `completedDate`
