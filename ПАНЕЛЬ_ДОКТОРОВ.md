# Панель докторов и ассистентов

## Обзор

Добавлен полный функционал для работы докторов и ассистентов в системе.

## Функционал для докторов

### 1. Расписание приемов
- Календарь с навигацией по дням (вчера/сегодня/завтра)
- Просмотр записей по часам (с 8:00 до 20:00)
- Фильтрация только консультаций (без операций)
- Статусы записей с цветовой индикацией
- Быстрый доступ к медицинской карточке

### 2. Операции и процедуры
- Отдельное расписание для операций и длительных процедур
- Календарь с навигацией по дням
- Детальная информация о каждой операции:
  - Длительность процедуры
  - Стоимость
  - Предоплата
  - Комментарии
- Открытие медицинской карточки

### 3. Мои пациенты
- Просмотр только своих пациентов (пациенты, у которых были записи к этому доктору)
- Поиск по имени и телефону
- Просмотр истории приёмов каждого пациента
- Открытие медицинских карточек

### 4. Медицинские карточки
- Заполнение диагноза
- Указание кода МКБ-10
- Запись рецептов и рекомендаций
- Внесение витальных показателей
- Назначение даты следующего визита

### 5. Управление услугами
- Создание собственных услуг (операций)
- Установка индивидуальных цен на услуги
- Выбор из существующих услуг клиники
- Редактирование и удаление своих услуг

## Функционал для ассистентов

### 1. Выбор доктора
- При создании ассистента владелец выбирает доктора для ассистирования
- Ассистент привязывается к конкретному доктору

### 2. Полный доступ доктора
Ассистент получает все права своего доктора:
- Просмотр расписания приемов доктора
- Просмотр расписания операций и процедур
- Доступ ко всем пациентам доктора
- Открытие и заполнение медицинских карточек
- Просмотр услуг доктора
- Создание записей и направлений

### 3. Ограничения
- Ассистент не может редактировать услуги доктора
- Видит только данные своего доктора

## Структура базы данных

### Изменения в таблице users
```sql
assistingDoctorId TEXT -- ID доктора для ассистента
```

### Новая таблица doctor_services
```sql
CREATE TABLE doctor_services (
  id TEXT PRIMARY KEY,
  doctorId TEXT NOT NULL,
  serviceId TEXT NOT NULL,
  customPrice DECIMAL(10,2), -- Индивидуальная цена
  isActive BOOLEAN DEFAULT true,
  createdAt TIMESTAMP DEFAULT NOW(),
  UNIQUE(doctorId, serviceId)
)
```

## API Endpoints

### Услуги докторов

#### GET /api/doctor-services
Получить услуги доктора
```
Query params:
- doctorId: ID доктора (опционально, для доктора автоматически берется его ID)
```

#### POST /api/doctor-services
Создать услугу доктора
```json
{
  "serviceId": "uuid",
  "customPrice": 50000 // опционально
}
```

#### PATCH /api/doctor-services/[id]
Обновить услугу доктора
```json
{
  "customPrice": 60000,
  "isActive": true
}
```

#### DELETE /api/doctor-services/[id]
Удалить услугу доктора

## Права доступа (RBAC)

### Доктор
- `appointments:view:own` - Просмотр своих записей
- `appointments:create` - Создание записей
- `appointments:update` - Обновление записей
- `patients:view` - Просмотр пациентов
- `medical:view:own` - Просмотр своих медкарточек
- `medical:create` - Создание медкарточек
- `medical:update` - Обновление медкарточек
- `directions:create` - Создание направлений

### Ассистент
Получает все права доктора:
- `appointments:view:own`
- `appointments:create`
- `appointments:update`
- `patients:view`
- `medical:view:own`
- `medical:create`
- `medical:update`
- `directions:create`

## Использование

### Создание ассистента
1. Перейти в "Пользователи"
2. Нажать "Создать пользователя"
3. Выбрать роль "Ассистент"
4. Выбрать доктора из списка
5. Заполнить остальные поля
6. Сохранить

### Добавление услуги доктором
1. Войти как доктор
2. Перейти на вкладку "Мои услуги"
3. Нажать "Добавить услугу"
4. Выбрать услугу из списка
5. Указать индивидуальную цену (опционально)
6. Сохранить

### Работа с медицинской карточкой
1. Открыть запись пациента
2. Нажать "Открыть карточку"
3. Заполнить:
   - Диагноз
   - Код МКБ-10
   - Рецепт
   - Рекомендации
   - Витальные показатели
4. Сохранить

## Тестирование

1. Создайте доктора
2. Создайте ассистента и привяжите к доктору
3. Войдите как доктор и добавьте услуги
4. Создайте запись к доктору
5. Войдите как ассистент и проверьте доступ к записям и пациентам доктора
6. Откройте медицинскую карточку и заполните данные

## Безопасность

- Доктор видит только своих пациентов
- Ассистент видит только пациентов своего доктора
- Все действия логируются в audit log
- Проверка прав доступа на уровне API
