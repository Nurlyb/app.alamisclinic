# Панель докторов и ассистентов - Реализовано

## Что сделано

### 1. База данных

#### Обновлена модель User:
- Добавлено поле `assistingDoctorId` - для привязки ассистента к доктору
- Ассистент может быть привязан к одному доктору
- Доктор может иметь несколько ассистентов

#### Создана модель DoctorService (услуги докторов/операции):
- `name` - название операции
- `description` - описание
- `createdBy` - ID владельца, который создал
- `isActive` - активна ли услуга
- Создаются только владельцем

#### Создана модель DoctorServiceAssignment (назначенные операции):
- `patientId` - пациент
- `doctorId` - доктор, который назначил
- `serviceId` - ID услуги доктора
- `price` - цена, которую поставил доктор
- `notes` - примечания
- `status` - PLANNED, COMPLETED, CANCELLED
- `scheduledDate` - планируемая дата
- `completedDate` - дата выполнения

### 2. API Endpoints

#### `/api/doctor-services` (GET, POST)
- GET - получить все активные услуги докторов
- POST - создать новую услугу (только владелец)

#### `/api/doctor-services/[id]` (PUT, DELETE)
- PUT - обновить услугу (только владелец)
- DELETE - деактивировать услугу (только владелец)

#### `/api/doctor-service-assignments` (GET, POST)
- GET - получить назначения (доктор видит только свои, ассистент - своего доктора)
- POST - назначить операцию пациенту (доктор или ассистент)

#### `/api/users/[id]/assign-doctor` (PUT)
- Привязать ассистента к доктору

### 3. Страницы и компоненты

#### Страница "Мои пациенты" (`/doctor-patients`)
**Для докторов и ассистентов:**
- Список всех пациентов, которые были на приеме (статус ARRIVED или DONE)
- Поиск по имени, телефону, ИИН
- Карточка пациента с:
  - Личной информацией (телефон, ИИН, дата рождения, адрес)
  - Кнопкой "Назначить операцию"
  - Списком назначенных операций с ценами и статусами
  - Историей осмотров (медицинские записи)

#### Компонент AssignOperationForm
- Выбор операции из списка
- Указание цены операции
- Планируемая дата (необязательно)
- Примечания

#### Вкладка "Услуги докторов" в настройках
**Для владельца:**
- Создание названий операций
- Редактирование операций
- Деактивация операций
- Список всех услуг докторов

#### Компонент DoctorServiceForm
- Форма для создания/редактирования услуг докторов
- Название операции
- Описание (необязательно)

### 4. Навигация

Добавлена ссылка "Мои пациенты" в сайдбар для:
- Докторов
- Ассистентов

### 5. Права доступа

#### Доктор:
- Видит только своих пациентов (через записи)
- Может назначать операции своим пациентам
- Может ставить цену на операции
- Может писать диагнозы (уже было)
- Видит историю осмотров своих пациентов

#### Ассистент:
- Выбирает доктора, которого ассистирует
- Имеет весь доступ этого доктора:
  - Видит тех же пациентов
  - Может назначать операции от имени доктора
  - Может писать диагнозы
  - Видит ту же историю

#### Владелец:
- Создает названия услуг докторов (операций)
- Управляет услугами докторов
- Видит все данные

## Как использовать

### Для владельца:

1. Зайти в "Настройки"
2. Перейти на вкладку "Услуги докторов"
3. Нажать "Добавить услугу доктора"
4. Указать название операции (например: "Лапароскопия", "Эндоскопия")
5. Добавить описание (необязательно)
6. Сохранить

### Для доктора:

1. Зайти в "Мои пациенты"
2. Найти нужного пациента
3. Открыть карточку пациента
4. Нажать "Назначить операцию"
5. Выбрать операцию из списка
6. Указать цену
7. Указать планируемую дату (необязательно)
8. Добавить примечания (необязательно)
9. Сохранить

### Для ассистента:

1. Владелец должен привязать ассистента к доктору через API или интерфейс пользователей
2. После привязки ассистент видит всех пациентов своего доктора
3. Может делать все то же самое, что и доктор
