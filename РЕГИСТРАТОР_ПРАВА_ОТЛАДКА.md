# Отладка прав доступа для регистратора

## Проблема
Регистратор получает ошибку "недостаточно прав" при попытке создать запись.

## Решение - Инструменты отладки

### 1. Панель отладки прав
В правом нижнем углу страницы расписания появилась панель отладки с кнопками:

- **🔄 (Обновить)** - Обновить данные пользователя
- **🔍 (Проверить)** - Проверить права на сервере и в базе данных  
- **🔑 (Токен)** - Принудительно обновить токен с актуальной ролью из БД
- **🔄 (Перезагрузить)** - Полная перезагрузка страницы

### 2. Информация в панели отладки
Панель показывает:
- Текущую роль пользователя
- Информацию о JWT токене (роль в токене, срок действия)
- Права на клиенте (frontend)
- Права на сервере (backend) 
- Данные из базы данных
- Совпадение ролей между токеном и БД

### 3. Пошаговая диагностика

#### Шаг 1: Проверьте панель отладки
1. Убедитесь что роль = RECEPTIONIST
2. Проверьте что токен не истек
3. Нажмите кнопку 🔍 для проверки сервера

#### Шаг 2: Сравните данные
- **Клиент appointments:create** должно быть ДА
- **Сервер appointments:create** должно быть ДА  
- **Роли совпадают** должно быть ДА

#### Шаг 3: Если роли не совпадают
Нажмите кнопку 🔑 для обновления токена с актуальной ролью из БД

#### Шаг 4: Если проблема сохраняется
1. Нажмите 🔄 для перезагрузки страницы
2. Или выйдите и войдите заново

### 4. Добавлено логирование
В консоли сервера теперь выводится подробная информация о проверке прав:
- Роль пользователя
- Требуемые права
- Результат проверки

### 5. Новые API endpoints для отладки
- `GET /api/debug/permissions` - проверка прав пользователя
- `GET /api/debug/user` - сравнение данных токена и БД
- `POST /api/auth/refresh-role` - обновление токена с актуальной ролью

## Файлы изменены
- `components/debug/UserPermissionsDebug.tsx` - панель отладки
- `app/api/debug/permissions/route.ts` - проверка прав
- `app/api/debug/user/route.ts` - проверка данных пользователя
- `app/api/auth/refresh-role/route.ts` - обновление токена
- `lib/auth/middleware.ts` - добавлено логирование
- `lib/auth/rbac.ts` - добавлено логирование

## Использование
1. Откройте страницу расписания
2. В правом нижнем углу увидите панель отладки
3. Нажмите 🔍 для диагностики
4. При необходимости используйте 🔑 для обновления токена
5. Проверьте консоль браузера и сервера для подробной информации
