# Добавление возможности создания записей для регистратора

## Что изменилось

✅ **Регистратор теперь может создавать новые записи на прием**

### 1. Обновлены права доступа (`lib/auth/rbac.ts`)

**Добавлено право:**
```typescript
RECEPTIONIST: [
  'appointments:view:all',
  'appointments:create', // ← НОВОЕ ПРАВО
  'appointments:arrive',
  'appointments:update',
  // ... остальные права
],
```

### 2. Обновлен интерфейс расписания (`app/schedule/page.tsx`)

**Добавлены кнопки создания записи:**

1. **Главная кнопка "Новая запись"** в заголовке страницы
2. **Кнопки "Создать запись"** в свободных временных слотах
3. **Кнопки "Создать запись на это время"** для отмененных записей
4. **Кнопка "Создать запись"** в пустом состоянии

**Обновлено условие отображения:**

### 3. Обновлено описание роли

**Новое описание для регистратора:**
```
📋 Управление записями и прием оплаты за услуги
```

## Функционал регистратора теперь включает:

### ✅ Создание записей
- Кнопка "Новая запись" в заголовке
- Создание записи в свободных слотах
- Создание записи на место отмененных

### ✅ Управление записями
- Отметка прихода пациентов
- Перенос записей
- Отмена записей (с ограничениями по времени)

### ✅ Работа с оплатой
- Прием оплаты за консультации
- Прием оплаты за операции/услуги
- Выдача чеков

### ✅ Ограничения по времени
- Полные права в течение 3 часов после времени приема
- После 3 часов - только перенос записей

## Преимущества для регистратора

1. **Полный цикл работы** - от создания записи до приема оплаты
2. **Удобство** - не нужно просить оператора создать запись
3. **Эффективность** - может сразу записать пациента при обращении
4. **Контроль** - видит все записи и может управлять расписанием

## Безопасность

- Сохранены все существующие ограничения по времени
- Права доступа проверяются через систему RBAC
- Аудит всех действий регистратора ведется

Теперь регистратор имеет полный набор инструментов для работы с расписанием!
