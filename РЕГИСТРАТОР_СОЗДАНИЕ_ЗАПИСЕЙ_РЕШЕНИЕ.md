# Решение проблемы "недостаточно прав" у регистратора

## Проблема
Регистратор получает ошибку "недостаточно прав" при попытке создать запись, несмотря на то что право `appointments:create` добавлено в роль RECEPTIONIST.

## Причина
Проблема может быть связана с кэшированием данных пользователя в браузере или устаревшим JWT токеном, который содержит старую информацию о правах.

## Решение

### 1. Добавлен компонент отладки прав
- В режиме разработки в правом нижнем углу страницы расписания появится панель отладки
- Показывает текущую роль пользователя, права доступа и информацию о токене
- Есть кнопки для обновления данных пользователя

### 2. Добавлена функция обновления данных пользователя
- В store аутентификации добавлена функция `refreshUser()`
- Позволяет принудительно обновить данные пользователя с сервера

### 3. Проверка прав в rbac.ts
Убедились что роль RECEPTIONIST имеет право `appointments:create`:

```typescript
RECEPTIONIST: [
  'appointments:view:all',
  'appointments:create',  // ✅ Право добавлено
  'appointments:arrive',
  'appointments:update',
  'patients:view',
  'payments:create',
  'payments:view:all',
  'payments:receipt',
  'refunds:create',
  'notifications:view',
],
```

### 4. Убраны ограничения в интерфейсе
В `app/schedule/page.tsx` убраны проверки `user?.role !== 'RECEPTIONIST'` для кнопок создания записи.

## Инструкции для пользователя

### Если проблема сохраняется:

1. **Проверьте панель отладки** (в development режиме):
   - Убедитесь что роль = RECEPTIONIST
   - Убедитесь что appointments:create = ДА
   - Проверьте что токен не истек

2. **Обновите данные пользователя**:
   - Нажмите кнопку с иконкой обновления в панели отладки
   - Или нажмите кнопку 🔄 для полной перезагрузки страницы

3. **Перелогиньтесь**:
   - Выйдите из системы
   - Войдите заново
   - Это обновит JWT токен с актуальными правами

4. **Очистите кэш браузера**:
   - Откройте DevTools (F12)
   - Перейдите в Application/Storage
   - Очистите localStorage и sessionStorage
   - Перезагрузите страницу

## Файлы изменены
- `lib/auth/rbac.ts` - добавлено право appointments:create для RECEPTIONIST
- `app/schedule/page.tsx` - убраны ограничения для регистратора, добавлен компонент отладки
- `lib/store/auth.ts` - добавлена функция refreshUser()
- `components/debug/UserPermissionsDebug.tsx` - компонент отладки прав
- `lib/utils/jwt-debug.ts` - утилиты для работы с JWT токенами

## Проверка
После применения изменений регистратор должен видеть кнопку "Новая запись" и иметь возможность создавать записи без ошибок прав доступа.
