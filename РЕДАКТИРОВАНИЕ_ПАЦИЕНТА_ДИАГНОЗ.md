# Добавлено редактирование данных пациента в форме диагноза

## Что добавлено

В форму "Записать диагноз" добавлена возможность редактировать данные пациента прямо из карточки, без перехода на другую страницу.

## Функциональность

### Кнопка "Редактировать"
- Расположена в правом верхнем углу карточки пациента
- При клике переключает карточку в режим редактирования
- Иконка: Edit2 (карандаш)

### Режим редактирования

**Доступные для редактирования поля**:
1. **ФИО** (обязательное) - текстовое поле
2. **ИИН** (опциональное) - текстовое поле, макс. 12 символов
3. **Телефон** (обязательное) - текстовое поле
4. **Дата рождения** (опциональное) - date picker
5. **Пол** (опциональное) - выпадающий список (Мужской/Женский)
6. **Адрес** (опциональное) - текстовое поле

### Валидация
- ФИО: обязательное поле
- Телефон: обязательное поле
- ИИН: максимум 12 символов
- Остальные поля опциональные

### Кнопки управления

**В режиме просмотра**:
- "Редактировать" - переключает в режим редактирования

**В режиме редактирования**:
- "Отмена" - возвращает в режим просмотра без сохранения
- "Сохранить изменения" - сохраняет данные и возвращает в режим просмотра

- Desktop: 2 колонки (grid-cols-2)
- Mobile: 1 колонка (grid-cols-1)
- Все поля автоматически перестраиваются

## Технические детали

### State управление
```typescript
const [isEditingPatient, setIsEditingPatient] = useState(false);
```

### Формы
Используются две отдельные формы:
1. **medicalRecordForm** - для диагноза и примечаний
2. **patientEditForm** - для данных пациента

### API запросы

**Обновление пациента**:
```typescript
PUT /api/patients/${patientId}
```

**Тело запроса**:
```typescript
{
  fullName: string;
  iin?: string;
  dob?: string;
  gender?: 'MALE' | 'FEMALE';
  phone: string;
  address?: string;
}
```

### Обновление данных
После успешного сохранения:
1. Обновляется кэш пациента (`refetchPatient()`)
2. Инвалидируются записи (`invalidateQueries(['appointments'])`)
3. Показывается уведомление "Данные пациента обновлены"
4. Автоматический возврат в режим просмотра

## Преимущества

1. **Удобство** - не нужно переходить на другую страницу для редактирования
2. **Скорость** - быстрое исправление ошибок в данных
3. **Контекст** - редактирование в контексте записи диагноза
4. **Безопасность** - валидация обязательных полей
5. **UX** - плавное переключение между режимами

## Сценарий использования

1. Доктор открывает форму "Записать диагноз"
2. Видит данные пациента
3. Замечает ошибку в ИИН или отсутствие даты рождения
4. Нажимает "Редактировать"
5. Исправляет/дополняет данные
6. Нажимает "Сохранить изменения"
7. Данные обновляются, карточка возвращается в режим просмотра
8. Продолжает записывать диагноз

## Права доступа

Редактирование доступно всем, кто может записывать диагноз:
- Доктора (DOCTOR)
- Ассистенты (ASSISTANT)

Используется существующий API endpoint `/api/patients/[id]` с методом PUT.

## Обработка ошибок

- Валидация на клиенте через zod schema
- Показ ошибок под полями ввода
- Toast уведомления при успехе/ошибке
- Блокировка кнопки во время сохранения

## Интеграция
