# Решение проблемы создания записей

## Найденная проблема
```
POST http://localhost:3000/api/patients 403 (Forbidden)
AxiosError: Request failed with status code 403
```

## Причина
Регистратор пытался создать **новую запись с новым пациентом**, но у роли RECEPTIONIST не было права `patients:create` для создания новых пациентов.

### Последовательность действий:
1. Пользователь выбирает "Новый пациент" в форме записи
2. Заполняет данные пациента (ФИО, телефон, источник)
3. Нажимает "Создать запись"
4. Система сначала пытается создать пациента через `POST /api/patients`
5. **ОШИБКА 403** - нет права `patients:create`
6. Создание записи не происходит

## Исправление

### 1. Добавлено право `patients:create` для роли RECEPTIONIST
```typescript
// lib/auth/rbac.ts
RECEPTIONIST: [
  'appointments:view:all',
  'appointments:create',
  'appointments:arrive',
  'appointments:update',
  'patients:view',
  'patients:create', // ✅ ДОБАВЛЕНО
  'payments:create',
  'payments:view:all',
  'payments:receipt',
  'refunds:create',
  'notifications:view',
],
```

### 2. Обновлена панель отладки
Теперь показывает оба права:
- `appointments:create`
- `patients:create`

### 3. Обновлены тестовые endpoint'ы
- `/api/debug/permissions` - проверяет оба права
- `/api/test/appointment-creation` - тестирует оба права

## Логика работы формы записи

### Существующий пациент:
1. Пользователь ищет пациента по имени/телефону
2. Выбирает из списка
3. Создается только запись (`POST /api/appointments`)

### Новый пациент:
1. Пользователь переключается на "Новый пациент"
2. Заполняет данные пациента
3. **Сначала** создается пациент (`POST /api/patients`) ← ЗДЕСЬ БЫЛА ОШИБКА
4. **Затем** создается запись (`POST /api/appointments`)

## Права доступа по ролям

### RECEPTIONIST (регистратор):
- ✅ `patients:view` - просмотр пациентов
- ✅ `patients:create` - создание новых пациентов
- ✅ `appointments:create` - создание записей
- ✅ `appointments:view:all` - просмотр всех записей

### OPERATOR (оператор):
- ✅ `patients:view` - просмотр пациентов  
- ✅ `patients:create` - создание новых пациентов
- ✅ `appointments:create` - создание записей

## Проверка решения

### 1. Используйте панель отладки:
- Нажмите 🧪 для прямого теста прав
- Должно показать:
  - `appointments:create: true`
  - `patients:create: true`

### 2. Если права показывают false:
- Нажмите 🔑 для обновления токена
- Или перелогиньтесь

### 3. Проверьте создание записи:
- Выберите "Новый пациент"
- Заполните минимальные поля (ФИО, телефон, источник)
- Нажмите "Создать запись"
- Должно работать без ошибок

## Файлы изменены
- `lib/auth/rbac.ts` - добавлено право `patients:create` для RECEPTIONIST
- `components/debug/UserPermissionsDebug.tsx` - показывает оба права
- `app/api/debug/permissions/route.ts` - проверяет оба права
- `app/api/test/appointment-creation/route.ts` - тестирует оба права

## Заключение
Проблема была в отсутствии права `patients:create` у роли RECEPTIONIST. Регистратор должен иметь возможность создавать новых пациентов при записи на прием, что логично для его роли.

Теперь регистратор может:
- Создавать записи для существующих пациентов
- Создавать новых пациентов и записи для них
- Просматривать всех пациентов и записи
